Logic การประมวลผล
① parse Unread Manuals ต่อ user เป็นรายชื่อเอกสารค้างจริง
② lookup Document Base (trim + case-insensitive) → Published + Ack period → Due Date
③ คำนวณสถานะ Over due / In-Time ใหม่จากวันที่ปัจจุบัน (ไม่ใช้ค่า Invalid period ที่ติดมากับไฟล์)
④ join แผนกจาก users.xlsx (คอลัมน์ Description) → รวมยอดรายแผนก · CD+QD รวมเป็น CD/QD · ซ่อน Unassigned
⑤ เอกสารที่จับคู่ไม่ได้ → แจ้งเป็น "เอกสารใหม่" ใน Document Base tab (นับเป็น Unknown)

Guideline — วิธีเตรียมไฟล์สำหรับ Import

ขั้นตอนการ export ไฟล์ทั้ง 3 ไฟล์ที่ใช้ในแท็บ Import · คลิกที่รูปเพื่อขยาย

📌 เครื่องมือนี้ใช้ไฟล์ 3 ไฟล์: ① User Performance (จาก Webmanual) · ② Document Base data (จาก QA Share Drive) · ③ User (จาก Webmanual) — ไม่จำเป็นต้องอัปโหลดครบทุกครั้ง ไฟล์ที่ไม่ได้อัปโหลดจะใช้ข้อมูลเดิมที่เคย import ไว้

1 · วิธี Export ไฟล์ "User Performance" จาก Webmanual

ไฟล์ช่องที่ 1 ในแท็บ Import
1
ไปที่แท็บ Report แล้วเลือก "User Performance"
2
คลิกปุ่ม Refresh ที่มุมขวาล่าง เพื่อให้ข้อมูลเป็นปัจจุบัน
3
คลิก "Export"
4
จะได้ไฟล์ Excel "User Performance" พร้อมนำมา Import ในเครื่องมือนี้ได้ทันที

2 · วิธี Export ไฟล์ "User" จาก Webmanual

ไฟล์ช่องที่ 3 ในแท็บ Import
1
ไปที่แท็บ User แล้วเลือก "Manage User"
2
ติ๊กช่องตามภาพเพื่อเลือกผู้ใช้ทั้งหมด — ตรวจสอบว่าตัวกรองเป็น "Active users" เท่านั้น
3
หลังเลือกครบแล้ว คลิก Action ที่มุมซ้ายล่าง แล้วเลือก "Export User"
4
จะได้ไฟล์ Excel "User" พร้อมนำมา Import ในเครื่องมือนี้ได้ทันที

3 · วิธีดึงไฟล์ "Document Base Data"

ไฟล์ช่องที่ 2 ในแท็บ Import
1
เข้า Share Drive ของบริษัท "Quality Assurance" แล้วเลือกโฟลเดอร์ "No. 9 Document Control Center & FSDS"
2
เลือกโฟลเดอร์ "No. 3 Document Acknowledgement"
3
เลือกโฟลเดอร์ "Document Base data"
4
เปิดไฟล์ Excel "Document Base data" — อัปเดตรายการเอกสารและ cross-check กับ Webmanual ให้ตรงกันก่อน จึงค่อยนำไฟล์มา Import ในเครื่องมือนี้
